
# ربات تلگرامی مار و پله 🐍🎲

## ویژگی‌ها
- دکمه‌های تعاملی (Inline Keyboard) برای join، شروع، پرتاب تاس، وضعیت و پایان بازی — نیازی به تایپ دستور نیست
- تصویر گرافیکی واقعی از صفحه بازی (۱۰۰ خونه، نردبان‌های سبز، مارهای قرمز، موقعیت هر بازیکن با رنگ و حرف اول اسمش) که بعد از هر تاس و هر بار /board فرستاده می‌شه

## دستورات
- `/start` - راهنما
- `/newgame` - شروع بازی جدید (توی PV بلافاصله شروع می‌شه، توی گروه لابی می‌سازه)
- `/join` - پیوستن به بازی (فقط گروه)
- `/startgame` - شروع رسمی بعد از join همه (فقط گروه، حداقل ۲ نفر)
- `/roll` - انداختن تاس
- `/board` - نمایش وضعیت فعلی همه بازیکن‌ها
- `/endgame` - پایان دادن به بازی جاری

## اجرای لوکال (تست قبل از دیپلوی)
```bash
python -m venv venv
source venv/bin/activate   # ویندوز: venv\Scripts\activate
pip install -r requirements.txt
export BOT_TOKEN="توکن ربات از BotFather"
python bot.py
```

## دیپلوی روی Railway
1. این پوشه رو به یک ریپوی جدید توی GitHub push کن.
2. توی Railway: **New Project → Deploy from GitHub repo** و ریپو رو انتخاب کن.
3. توی تب **Variables** یک متغیر با نام `BOT_TOKEN` بساز و توکن ربات رو مقدارش بذار.
4. Railway به صورت خودکار `Procfile` رو تشخیص می‌ده و پروسه `worker` رو اجرا می‌کنه.
5. توی تب **Deployments** لاگ رو چک کن؛ اگه پیام "ربات در حال اجراست..." رو دیدی، همه چی درسته.

## نکته مهم درباره‌ی حافظه بازی‌ها
وضعیت بازی‌ها (موقعیت بازیکن‌ها) توی حافظه (RAM) نگه داشته می‌شه، نه دیتابیس.
یعنی اگه سرویس روی Railway ری‌استارت بشه (مثلاً بعد از دیپلوی جدید یا کرش)، بازی‌های در حال انجام از دست می‌رن.
برای بازی‌های ساده و کوتاه‌مدت مشکلی نیست؛ اگه بعداً خواستی پایدارتر بشه (با SQLite یا Redis)، بگو تا اضافه کنم.
