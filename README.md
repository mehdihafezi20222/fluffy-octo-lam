دیپلوی روی Railway
این پوشه رو به یک ریپوی جدید توی GitHub push کن.
توی Railway: New Project → Deploy from GitHub repo و ریپو رو انتخاب کن.
توی تب Variables یک متغیر با نام BOT_TOKEN بساز و توکن ربات رو مقدارش بذار.
Railway به صورت خودکار Procfile رو تشخیص می‌ده و پروسه worker رو اجرا می‌کنه.
توی تب Deployments لاگ رو چک کن؛ اگه پیام "ربات در حال اجراست..." رو دیدی، همه چی درسته.
نکته مهم درباره‌ی حافظه بازی‌ها
وضعیت بازی‌ها (موقعیت بازیکن‌ها) توی حافظه (RAM) نگه داشته می‌شه، نه دیتابیس.
یعنی اگه سرویس روی Railway ری‌استارت بشه (مثلاً بعد از دیپلوی جدید یا کرش)، بازی‌های در حال انجام از دست می‌رن.
برای بازی‌های ساده و کوتاه‌مدت مشکلی نیست؛ اگه بعداً خواستی پایدارتر بشه (با SQLite یا Redis)، بگو تا اضافه کنم.
